#+TITLE: OpenMediaVault Status Monitor
#+DESCRIPTION: Sistema di monitoraggio "Dead Man's Switch" per server OpenMediaVault tramite GitHub Pages e Healthchecks.io.
#+LANGUAGE: it

* Descrizione del Progetto
Questo repository ospita una soluzione leggera e statica per monitorare lo stato di attività (online/offline) di un server domestico basato su OpenMediaVault (OMV). Il sistema utilizza il paradigma del "Dead Man's Switch" (interruttore dell'uomo morto): il server notifica periodicamente un servizio di monitoraggio esterno; se le notifiche si interrompono, la pagina web ospitata su GitHub Pages aggiorna automaticamente il proprio stato mostrando una schermata di errore.

* Architettura del Sistema
L'infrastruttura si compone di tre elementi principali:
1. **OpenMediaVault (Server)**: Esegue un'attività pianificata ogni minuto che invia un segnale HTTP (ping via cURL) all'endpoint di controllo.
2. **Healthchecks.io (Servizio di Monitoraggio)**: Riceve i ping dal server. Se non riceve aggiornamenti entro la tolleranza stabilita, contrassegna lo stato del server come "down" e aggiorna un badge JSON pubblico.
3. **GitHub Pages (Interfaccia Utente)**: Una pagina HTML statica che interroga asincronamente il badge JSON di Healthchecks.io tramite JavaScript (Fetch API) e aggiorna l'interfaccia utente in tempo reale senza ricaricare la pagina.

* Configurazione del Sistema

** 1. Configurazione su Healthchecks.io
1. Creare un account su [[https://healthchecks.io/][Healthchecks.io]].
2. Creare un nuovo controllo (Check).
3. Impostare i seguenti parametri temporali:
   - **Period (Periodo)**: 1 minuto
   - **Grace (Tolleranza)**: 2 o 3 minuti (per prevenire falsi positivi dovuti a latenze di rete).
4. Copiare l'URL di ping fornito (es. =https://hc-ping.com/tuo-uuid-segreto=).
5. Nella sezione "Badges" del progetto, abilitare l'accesso pubblico e copiare l'URL del formato **JSON**.

** 2. Configurazione su OpenMediaVault
Invece di utilizzare direttamente il crontab di sistema, si raccomanda l'uso dell'interfaccia web di OMV per garantire la persistenza delle impostazioni durante gli aggiornamenti del firmware.

1. Accedere al pannello di controllo web di OMV.
2. Navigare su *Sistema* > *Attività pianificate*.
3. Cliccare su *Aggiungi* (+) e configurare come segue:
   - **Abilita**: Attivo
   - **Utente**: =root=
   - **Comando**:
     #+BEGIN_SRC bash
     curl -fsS -m 10 --retry 5 -o /dev/null https://hc-ping.com/tuo-uuid-segreto
     #+END_SRC
   - **Invia email**: Disattivato (per evitare la saturazione della casella di posta).
   - **Pianificazione**: Impostare il minuto su =*= (Ogni minuto) e lasciare gli altri campi su =*=.
4. Salvare e applicare le modifiche tramite la notifica di sistema in alto.

** 3. Distribuzione su GitHub Pages
Il file =index.html= in questo repository gestisce autonomamente il recupero dello stato e la visualizzazione grafica.

1. Caricare il file =index.html= nel ramo principale (=main= o =master=) di questo repository.
2. Modificare la variabile =badgeJsonUrl= all'interno dello script incorporato in =index.html= inserendo l'URL JSON ottenuto da Healthchecks.io.
3. Accedere alle *Settings* del repository GitHub, navigare nella sezione *Pages* e abilitare il deployment dal ramo selezionato.

* Struttura dei File
- =index.html=: Pagina di atterraggio principale contenente la logica JavaScript per il controllo dello stato, gli stili CSS incorporati e le risposte visive per gli stati Online, Offline ed Errore di rete.
- =README.org=: Questa documentazione tecnica.

* Risoluzione dei Problemi
- **Il server risulta offline ma è acceso**: Verificare i log di OMV per assicurarsi che il comando cURL venga eseguito senza errori di rete locali o problemi di risoluzione DNS.
- **Latenza nell'aggiornamento della pagina**: Il client esegue un polling automatico del badge JSON ogni 60 secondi. È possibile forzare l'aggiornamento ricaricando la pagina del browser.
